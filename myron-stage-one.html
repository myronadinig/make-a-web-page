Important Concepts

Stage 0: The Basics of the Web and HTML

What is the World Wide Web?

The World Wide web is a collection of HTML documents. HTML stands for Hypertext Markup Language. Links between pages are called Hyperlinks. There are many types of files that can be found on the web.

These include:

  • - Plain text
  • - HTML
  • - images
  • - videos
  • - and music
HTML

HTML is made up of

  • - text content (what you see)
  • - markup (what it looks like)
  • - references to other documents (e.g. images & videos)
Tags and Elements

HTML documents are also made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Stage 1: Make your own Webpage

HTML & CSS

HTML and CSS are both "languages".
HTML controls the "structure"of a web page.
CSS controls the "style" of a page (how it looks).
When programmers talk about the "DOM" (Document Object Model). they are talking about the tree-like structure of a page.

CSS stands for "Cascading Style Sheets" (specific syntax + rules - to control look of html page). Inheritance is a key feature in CSS, it relies on the ancestor-descendant relationship to operate. You can have several css files.

The "tree-like structure" of HTML

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. My mother had multiple children. So did her mother, and so on...

In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

Boxifying a Design

Everything on the web is boxes. You define boxes with div tags.

Code, Test, and Refine

Webdesign is an iterative process.

Typically the process is:

1. Get the structure right. (by visually boxifying)
2. Then work on tags.
3. Size the boxes.
4. Position the boxes.

You also want to:

1. Look for natural boxes.
2. Look for repeated styles & semantic elements.
3. Write your HTML.
4. Apply styles (from biggest to smallest).
5. Fix things.

Avoiding Repetition

Avoiding repetition is important for a variety of reasons.

Avoiding Errors: When programmers don't avoid repetition, they will often have to do the same thing over and over. This may mean copy and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times. This can lead to errors when the programmer decides to make a change to something. If they don't dilligently make that same change everywhere the repeated code appears, problems will arise.
